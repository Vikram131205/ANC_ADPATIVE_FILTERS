# Python Adaptive Signal Processing Handbook

Theoretical Foundation: Active Noise Cancellation via Adaptive FilteringTo move beyond basic signal addition, this project explores the stochastic process of noise cancellation. In a real-world acoustic environment, noise is not a static mathematical entity; 

it is a dynamic signal that undergoes transformation as it travels through space.1. The Physics of Destructive InterferenceAt its core, ANC relies on the principle of superposition. If two sound waves of the same frequency and amplitude are $180^\circ$ out of phase, they interfere destructively, resulting in a net pressure of zero.

$$p_{total}(t) = p_{noise}(t) + p_{anti-noise}(t) \approx 0$$

In practice, the challenge is that the "anti-noise" must perfectly match the noise at the exact point of the listener's ear, accounting for delays, echoes, and hardware latency.2. Adaptive Filter ArchitectureAn adaptive filter is a self-learning system that adjusts its coefficients (weights) to minimize a specific cost function. Unlike a standard FIR filter with fixed coefficients, an adaptive filter "evolves" to track non-stationary signals.The Three Key Components:Filter Structure: Usually an FIR (Finite Impulse Response) structure.Performance Criterion: The Mean Squared Error (MSE), defined as $J(n) = E[e^2(n)]$, where $e(n)$ is the difference between the desired signal and the filter output.Adaptive Algorithm: The logic used to update the weights.

LMS/NLMS TheoryThe LMS algorithm updates weights based on the instantaneous gradient:$$\mathbf{w}(n+1) = \mathbf{w}(n) + \mu e(n) \mathbf{x}(n)$$NLMS improves this by normalizing the step size $\mu$ by the power of the input signal, preventing the filter from "diverging" when the noise gets too loud.RLS TheoryRLS treats the problem as a least-squares minimization. It uses the entire history of the signal to find the optimal weights, making it incredibly fast at converging but much more "expensive" for a processor to calculate.4. Acoustic Path ModelingIn a professional ANC setup, we must model two distinct physical paths:The Primary Path ($P(z)$): The transfer function of the environment. It represents how the noise source (e.g., an engine) sounds by the time it reaches the user's ear.The Secondary Path ($S(z)$): The transfer function of the ANC hardware itself—the DAC, the amplifier, the speaker, and the physical distance between the speaker and the error microphone.An advanced project (like this implementation) attempts to "identify" these paths using the filter to ensure the anti-noise arrives at the ear exactly when the noise does.5.
